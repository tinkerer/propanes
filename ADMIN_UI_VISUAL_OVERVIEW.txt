TAB ID FORMAT
=============
Tab IDs are strings in format: '<type>:<identifier>'

  TYPE                IDENTIFIER          RENDERED BY
  ────────────────────────────────────────────────────────────────
  jsonl               sessionId            <JsonlView />
  feedback            sessionId            <FeedbackCompanionView />
  terminal            sessionId            <TerminalCompanionView />
  iframe              sessionId            <IframeCompanionView />
  isolate             componentName        <IsolateCompanionView />
  url                 fullUrl              <IframeCompanionView />
  admin-assist        context              <AdminAssistCompanion />  ← NEW

HOW TO ADD ADMIN CHAT BOX
=========================

Step 1: Update lib/sessions.ts
  ├─ Add 'admin-assist' to CompanionType union
  ├─ Update extractCompanionType() to recognize 'admin-assist:'
  └─ Update renderTabContent() switch to handle 'admin-assist'

Step 2: Create components/AdminAssistCompanion.tsx
  ├─ Chat message list
  ├─ Input textarea (Cmd+Enter to submit)
  ├─ API call: api.setupAssist({ request, entityType: 'admin' })
  ├─ Load messages from returned sessionId
  └─ Message rendering (reuse StructuredView or MessageRenderer)

Step 3: Update GlobalTerminalPanel.tsx
  ├─ Import AdminAssistCompanion
  ├─ Auto-open 'admin-assist:welcome' tab on first load
  └─ Add case in renderTabContent():
       case 'admin-assist': return <AdminAssistCompanion tabId={tabId} />;

Step 4: Update app.css (optional)
  └─ Add styles following .ai-assist-* pattern

  ├─ Small popover for page-specific AI help
  ├─ Pattern: button → popover → textarea → submit → open session
  └─ Used on: Feedback detail, Aggregate, App settings

SetupAssistButton.tsx
  ├─ Draggable popover for infrastructure setup
  ├─ Pattern: button → draggable popover → preset buttons
  ├─ Has presets for: machine, harness, agent, sprite
  └─ Used on: Infrastructure page, Settings pages

AddAppModal.tsx
  ├─ Good FSM pattern: null → mode selection → form → success
  ├─ Good form + error handling patterns
  └─ Good copy-to-clipboard pattern

JsonlView.tsx
  ├─ Loads JSONL files from API
  ├─ Two parsers: JsonOutputParser + TerminalOutputParser
  ├─ Three view modes: Terminal, Structured, Split
  └─ Message filtering via dropdown

StructuredView.tsx
  ├─ Group messages by type
  ├─ Show token count and tool count per group
  ├─ Click to expand/collapse
  └─ Calls MessageRenderer for individual messages

MessageRenderer.tsx
  ├─ Renders 15+ tool types
  ├─ Syntax highlighting with Prism
  ├─ Image lightbox with base64/URL detection
  ├─ Auto-truncate long output with expand
  └─ Tool-specific formatting (Bash, Edit, Grep, etc.)


KEY STATE FUNCTIONS
===================

  toggleCompanion(sessionId, type)

  openUrlCompanion(url)
    └─ Opens a URL in iframe companion

  closeTab(tabId)
    └─ Closes a tab

  persistPanelState()

SUMMARY
=======
The admin UI is well-architected with:
  • Clear routing via URL
  • Consistent component structure
  • Centralized state management
  • Reusable patterns for forms, modals, popovers
  • Extensible companion tab system

Adding an admin chat box is a straightforward 4-step process:
  1. Define new companion type in state
  2. Create component to render it
  3. Wire it into GlobalTerminalPanel
  4. Auto-open on first load

No layout changes needed. It integrates naturally as another tab type.
